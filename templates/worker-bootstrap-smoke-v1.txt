For this smoke profile, ALWAYS invoke the orchestration CLI via the exact `{{orc_bin}}`

When a TASK_START block arrives:
  1. cd into the `assigned_worktree` path from the payload.
  2. Call {{orc_bin}} run-start --run-id=<run_id> --agent-id={{agent_id}} immediately.
  3. Signal explore phase and read the markdown task spec at `task_spec_path`:
       {{orc_bin}} progress --event=phase_started --phase=explore --run-id=<run_id> --agent-id={{agent_id}}
  4. Signal implement phase, make only the requested changes, and run `npm test`:
       {{orc_bin}} progress --event=phase_started --phase=implement --run-id=<run_id> --agent-id={{agent_id}}
     Gate: `npm test` MUST exit 0 before you continue.
  5. Signal review phase and create a single task commit:
       {{orc_bin}} progress --event=phase_started --phase=review --run-id=<run_id> --agent-id={{agent_id}}
       git add .
       git commit -m "feat(smoke): complete {{agent_id}} task"
     Do NOT spawn sub-agents in this smoke profile.
  6. Signal complete phase and update the task markdown in this assigned worktree to `status: done`:
       {{orc_bin}} progress --event=phase_started --phase=complete --run-id=<run_id> --agent-id={{agent_id}}
       - Edit the task spec file referenced by `task_spec_path`
       - Change its frontmatter `status:` field to `done`
       - Stage that spec change and fold it into the task commit:
           git add <task_spec_path>
           git commit --amend --no-edit
       - Do not call `{{orc_bin}} task-mark-done` from the worker session; the coordinator marks shared runtime state done after merge
  7. Emit a heartbeat, rebase onto main, then emit another heartbeat and hand off:
       {{orc_bin}} run-heartbeat --run-id=<run_id> --agent-id={{agent_id}}
       git rebase main
       {{orc_bin}} run-heartbeat --run-id=<run_id> --agent-id={{agent_id}}
       {{orc_bin}} run-work-complete --run-id=<run_id> --agent-id={{agent_id}}
  8. Signal finalize phase and wait for coordinator follow-up when it exists:
       {{orc_bin}} progress --event=phase_started --phase=finalize --run-id=<run_id> --agent-id={{agent_id}}
       - If finalization succeeded or the session can stop, call:
           {{orc_bin}} run-finish --run-id=<run_id> --agent-id={{agent_id}}
       - Do not merge to main, delete the worktree, or delete the branch yourself.

If you hit an unrecoverable blocker, call:
